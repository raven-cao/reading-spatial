ML & AMICI

AMICI

Attention-based Model for Inferring Cell–Cell Interactions

What problem does it solve?

Prior tools like CellChat infer cell–cell communication by (1) averaging expression across all cells of a type, and (2) checking a curated ligand–receptor database. This has two fundamental limits: spatial information is discarded, and anything outside the database is invisible.

AMICI asks a different question entirely: can a cell's gene expression be predicted from its spatial neighbors' expression? If yes, those neighbors are influencing it — no database needed.

Key insight
AMICI doesn't verify that a ligand-receptor pair can bind. It detects whether a neighbor's presence actually changes the receiver cell's expression state. The difference between "possible" and "observed."

The five-stage pipeline

1. Preprocessing — Cell segmentation (assign mRNA signals to cells), cell type annotation (marker gene → Leiden clustering → human labels), filter to highly variable genes, normalize to 1000 total counts per cell, log1p transform to compress the distribution.

2. Define neighbors — For each receiver cell, find the 50 nearest neighbors by Euclidean distance. Critically: same cell type neighbors are excluded. This prevents the model from taking the shortcut of copying nearby cells of the same type.

3. Self-supervised training (core) — Mask the receiver's expression vector. Let the model see only: (a) receiver cell type label, (b) 50 neighbors' expression vectors, (c) distance to each neighbor. Train it to reconstruct the masked expression. The correct answer is already in the data — no human labels needed. This is what "self-supervised" means.

4. Attention weights — Each neighbor gets a weight computed as:

weight = softmax(b₀ − b₁ × distance)

where b₀ = learned relevance between receiver type and neighbor expression, b₁ = distance decay coefficient (forced positive, so distance always reduces weight). A "null token" absorbs weight when no neighbor is actually relevant.

5. Downstream interpretation — Attention weights tell you who influences whom. Ablation (masking one sender type entirely and measuring reconstruction drop) tells you which genes are regulated by that interaction. The spatial distribution of weights tells you at what distance the interaction is effective (length scale).

Why "dynamic" and "interpretable"?

Dynamic: Every cell has its own attention profile. Two CD8+ T cells in different locations get different weights — one exhausted by surrounding tumor cells, one activated near dendritic cells. CellChat gives one number per cell-type pair; AMICI gives one profile per individual cell.

Interpretable: AMICI uses a single attention layer. The weight on each neighbor is directly readable: "67% of this T cell's deviation from its type average is attributable to that tumor cell 20µm away." You can design an experiment around that. A deep black-box model can't give you that.

Analogy (toolbox version)
Each cell is a worker with a toolbox of 6,000 tools. Their toolbox usage pattern reflects their state. AMICI asks: can I predict which tools worker A is using today, given the toolbox usage of the 50 nearest colleagues from other departments? If I can — those colleagues are influencing A's work. The attention weights tell me which colleague mattered most, and ablation tells me which specific tools they influenced.

What AMICI is not

It is not a causal proof. Predicting B from A is stronger than correlating A and B (CellChat), but weaker than a knockout experiment. Confounders exist — two cells sharing a neighbor may both respond to the same upstream signal, making them appear to influence each other when neither does. AMICI narrows the hypothesis space; experiments close it.

My question / remaining uncertainty
The model uses receiver cell type as an embedding input (b₀). This is learned during training. But what exactly is being encoded in that embedding? Is it something like "T cells are sensitive to PD-L1-expressing neighbors"? And how does this interact with the exclusion of same-type neighbors?
