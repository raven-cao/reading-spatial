Attention Mechanism

Weighted aggregation of relevant information from a set of sources

The core idea

Attention is a weighted sum. Given a target (the receiver cell), and a set of sources (neighbor cells), compute a weight for each source based on how relevant it is to the target. Then aggregate the source information proportionally to those weights.

The formula in AMICI: weight = softmax(b₀ − b₁ × distance)

  • b₀: learned relevance — does this type of receiver respond to this neighbor's expression profile?
  • b₁ × distance: spatial decay — closer neighbors get higher weight. b₁ is forced positive so distance can only reduce weight, never amplify it.
  • softmax: normalizes all weights to sum to 1, and amplifies differences (via exponentiation).

What softmax actually does

Raw scores might be [8, 3, 5, 1, 2]. Softmax: take e^x for each, divide by sum of all e^x. Result ≈ [0.84, 0.02, 0.11, 0.003, 0.008]. The highest score gets a dominant share. This makes the model commit to a sparse, interpretable allocation of attention.

Why not just use raw similarity scores?
Raw scores don't sum to 1, making them hard to interpret as "contribution." Softmax turns them into a probability distribution over neighbors — each weight is the fraction of the receiver's reconstruction attributable to that neighbor.

Multi-head attention

AMICI runs multiple attention heads in parallel. Each head learns a different distance scale. Head 1 might specialize in 0–25µm (contact-dependent signaling), Head 2 in 0–100µm (paracrine diffusion). Biologically, this reflects that different signaling mechanisms operate at different ranges.

Why high similarity ≠ high influence

Two cells can have similar expression because they share the same microenvironment (both are responding to a third signal), not because one is influencing the other. Attention weight is based on predictive relevance learned from training, not raw cosine similarity. A neighbor that consistently co-occurs with certain receiver states gets high weight even if their expression patterns are complementary rather than similar.

Office analogy
b₀ measures: "every time a legal dept person with this kind of workload sits near an engineer, the engineer's tool usage changes in a predictable way." It's a learned historical pattern, not a similarity score. Lisa and the engineer don't use the same tools — but Lisa's presence predicts which tools the engineer picks up next.
